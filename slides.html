The Problem


AI agents can run ML experiments autonomously.

But why would anyone donate their GPU?


We need an incentive mechanism that rewards agents proportional to their research contributions.

What is Autoresearch?


Give an AI agent a small LLM and let it experiment autonomously.


Modify code → Train → Check val_bpb → Keep or discard → Repeat


"The next step for autoresearch is that it has to be asynchronously massively collaborative for agents (think: SETI@home style)." — @karpathy, March 2026

The Experiment Loop


  1. THINK — check what the swarm tried, pull global best
  2. CLAIM — register experiment to avoid duplicates
  3. EDIT — modify train.py (arch, optimizer, hyperparams)
  4. TRAIN — run a fixed-budget experiment
  5. PUBLISH — share results + insights with the swarm
  6. LOOP — repeat forever

The Protocol: ExperimentResult


@dataclass
class ExperimentResult:
    agent_id: str = ""
    val_bpb: float = 0.0          # the key metric
    memory_gb: float = 0.0
    training_seconds: float = 0.0
    total_tokens_M: float = 0.0
    num_steps: int = 0
    num_params_M: float = 0.0
    status: str = ""              # "completed" | "failed"
    description: str = ""
    experiment_key: str = ""
    train_py_hash: str = ""       # SHA256 for dedup
    timestamp: float = 0.0
    accepted: bool = False        # set by validator
    score: float = 0.0            # set by validator

Demo: plain dataclass over HTTP/JSON. Production: bt.Synapse over dendrite/axon.

The Miner


HTTP server that runs experiments on demand.

Validator sends POST /run → miner runs train_lite.py

Parses output, returns ExperimentResult as JSON.


One experiment at a time (lock-guarded). GET /status shows idle/running.

Miner Internals


def _run_experiment(self) -> ExperimentResult:
    """Run train_lite.py and return the parsed result."""
    env["AUTORESEARCH_TIME_BUDGET"] = str(self.server.time_budget)

    proc = subprocess.run(
        [sys.executable, "train_lite.py"],
        capture_output=True, text=True,
        timeout=self.server.time_budget + 60,
        env=env,
    )

    metrics = parse_train_output(proc.stdout)
    return ExperimentResult(
        agent_id=miner_id,
        val_bpb=metrics["val_bpb"],
        train_py_hash=hash_file("train_lite.py"),
        status="completed",
        ...
    )

Scoring Formula


Lower val_bpb = better model = higher score


Best miner: score = 1.0

Other miners: score = (best / miner_bpb)²


Scores normalized to weights that sum to 1.0

Anti-Cheat


CheckRule
Deduptrain_py_hash — first submitter wins
StalenessResults older than 1 hour are ignored
Sanity boundval_bpb ≥ 0.5 (rejects bogus scores)
Status checkOnly "completed" results score

Prevents copying others' work, submitting stale results, or gaming metrics
